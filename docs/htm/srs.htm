General Information
This document will describe the overall requirements of the Intel 8085 Simulator. The document will follow the IEEE guidelines for requirement specifications with some variations.

General Information
The product to be designed is an Intel 8085 Simulator. The product is developed with a target for minimal deployment and requirement specifications, as will be observed from the document. It is targeted as a testing and development platform for assembly language programming on Intel 8085 processor, but on moderm PCs and Laptop.
Purpose
The software to be developed is a Intel 8085 Simulator. It is targeted to run on modern Desktops and Laptops, with as minimal system configuration as possible. It will be a benifit to establish cross-platform support for the final executable, as far as possible. For this reason, we will be using Microsoft .NET Framework 2.0. This will benifit users to run the product on all Windows OS with Microsoft .NET framework 2.0 installed. And the product will also execute on other OS with MONO Framework installed. But the execution on non-Windows OS is not guarenteed.
Scope
The final software is developed to benifit 8085 developers, to test their assembly language or machine code based on Intel 8085 microprocessor. The product will target the audience which includes Students from Computer Science, Electronics, etc. who are in phase of learning the basics of Assembly language programming. This product will also be benificiary for those who cannot get their hands on an Intel 8085 SDK, but have access to any modern computing systems.

Current Functional Procedure
  • Existing systems comparable to the Intel 8085 Simulator designed are GNU8085 Simulator, 8085 Simulator by J.Tech. Softwares, 8085 Simulator hosted at http://8085simulator.codeplex.com/, . However, there are several limitations in the existing simulators. These are either commercial in version, or have limited features. Hence, the target is to eliminate the limitations as far as possible, alongwith keeping the software as freeware.
  • The system designed here is a development platform, and requires no maintenance.
  • The application had been developed by an individual, as this is designed for MCA project work and to facilitate academic students.
  • The input data for the application is the "Source Assembly code", output is binary data in Intel 8085 understandable format, loaded into the simulator memory.
  • The execution and visual output is obtained within the working IDE of the software application.
Functional Objectives
  • The application had been developed to facilitate academic students.
  • The main objective is to make students aware of 8085 assembly language development.
  • The application is a freeware and is always open-source.
  • It provides least execution cost than buying a 8085 simulator.
Performace Objective
  • The application is a freeware and is always open-source.
  • It provides least execution cost than buying a 8085 simulator.
  • It is made available online at sourceforge.net. Hence, is widely available without much hassle.
  • The speed of the program is however limited by the execution system and present load.
  • Compared to average 8085 SDKs available, it provides enhanced control over Instruction execution clock, to facilitate effective debugging.
Evaluation Criteria
  • The development time was about 6 month, which included studies conducted, code writing and UI design, testing and execution report (monitored by Author and also reports verbally communication be users, which mostly included students).
  • The system is fairly easy to use. But requires programming knowledge of 8085.
